| Stage  | R (Ω) | L (µH) | τ = L/R (µs) | Pulse (µs) | Pulse/τ | I_peak = V/R (A) | Verdict                            |
|--------|-------|--------|--------------|------------|---------|------------------|------------------------------------|
| Coil 1 | 1.3   | 476    | 366          | 2600       | **7.1** | 47.7             | **Over-driven** — pulse ≫ 3τ; saturation reached at ~1100 µs, remainder is resistive heating and extends suck-back window |
| Coil 2 | 2.8   | 1900   | 679          | 500 *(top-quartile winner)* | **0.74** | 22.1 | Below 1τ — below "classical" 1–3τ window, but top-quartile profile and single best run both favour it over 1200 µs (1.77τ). Suggests suck-back dominates at this stage (see note below). |
| Coil 3 | 5.0   | 1000   | 200          | 6300       | **31.5** | 12.4             | **Massively over-driven** — but coil 3 is not physically attached per project memory; field-test this later. |

**Note on coil 2 "sub-τ beats supra-τ"**: the projectile is moving when coil 2 fires. Keeping the field on longer means it is still pulling when the projectile has already left the magnetic centre, creating suck-back (retarding force). The optimum therefore is not a τ-based saturation match; it is whatever on-time delivers peak force during the "before-centre" half and turns off near the pole. The data showing 500 µs (~52 % I_sat, ~11.5 A) beating 1200 µs (~83 % I_sat, ~18.3 A) is consistent with this physics: extra current gains less than extra suck-back costs.

Because ratings do not vary across runs, velocity correlation with pulse/τ is identical to pulse_µs in this dataset — no dataset-level discrimination possible. Correlation is the same signal expressed in different units.

## Recommendations

### Optimal Config (Best Known)

- **`gate_1_coil_2_delay_us = 110`** — weaker signal (\|t\| = 1.69) but no value in the tested set outperforms it; treat 110 as a tentative centre for a finer sweep rather than a settled optimum.

All other parameters unchanged from current (single-value in dataset → no evidence to move).

### Next Field Test Plan

**Priority 0 — diagnose gate-detection failures first.** 10.8 % valid-G1→G2 on the active sequence is session-breaking. If 9 of 10 runs don't clear G2 cleanly, the "optimum" is whatever config happens to produce a clean detection, not whatever produces peak velocity. Investigate mechanical/optical/wiring state before more sweeps.

Once detection is back to normal (~90 %+ valid), the following runs target the remaining unknowns:

1. **Coil 1 pulse sweep:** 2600 → trim toward saturation point.
   - Values: **{800, 1100, 1500, 2000, 2600}** µs. Target pulse/τ = 2.2, 3.0, 4.1, 5.5, 7.1. Hypothesis: with τ = 366 µs, anything past ~3τ = 1100 µs is resistive heating and suck-back extension only; shorter should equal or beat 2600 µs and reduce coil heat.
   - At least 3 runs per value (prior session established ≥3 for noise floor).

2. **Coil 2 pulse fine sweep around the optimum:**
   - Tested so far: {500, 1200}. Gap is 700 µs wide; real optimum may be 300 µs or 600 µs.
   - Values: **{300, 400, 500, 600, 800}** µs. 3+ runs each.

3. **Gate 1 → Coil 2 delay fine sweep:**
   - Tested so far: {0, 10, 110, 1000, 3000}. Massive gaps around the apparent 110 µs optimum.
   - Values: **{50, 80, 110, 140, 180, 250}** µs. 3+ runs each.

4. **Rail-source-active variation (untested):**
   - All runs at 0. With a 62 V rail available (v_coil_ceiling) the cap bank can be held at ceiling between runs. Rerun the current best config with `rail_source_active = 62` to see if held-ceiling vs decayed-cap makes a difference for back-to-back runs.

5. **Coil-rating experiment (requires hardware swap):**
   - All coil R/L values in the DB are defaults. If the operator has a spare coil with different R/L, try swapping into stage 2 and re-running the pulse sweep. Specifically, a *lower-L coil* at stage 2 (say L ≈ 800 µH, R ≈ 2 Ω → τ ≈ 400 µs) should let a longer on-time (~1τ ≈ 400 µs) hit the same current fraction as 500 µs on the current 679 µs-τ coil — a direct test of the "suck-back limits on-time" hypothesis above.
   - Also: if ratings are edited per-snapshot, the top-quartile analysis can actually *use* the pulse/τ ratio as a predictor in the next skill run.
